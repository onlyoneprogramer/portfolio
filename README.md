# 이진호 | 보안·시스템 관점에서 0→1을 만드는 소프트웨어 엔지니어

> **Top-down으로 사고하고, Bottom-up으로 검증하며,  
> 검증 결과를 구조로 고도화하는 엔지니어**

저는 시스템을 **Top-down으로 이해**합니다.  
요구사항·위협·운영·확장까지 전체 흐름을 먼저 머릿속에 그린 뒤,  
실제 개발은 **가장 불확실한 하위 레벨부터 Bottom-up으로 PoC를 만들어 검증**합니다.

- **사고**: 큰 그림(아키텍처·리스크·경계)을 먼저 정의  
- **구현**: 실제 동작하는 최소 단위부터 만들고, 검증 결과로 구조를 반복 개선  

이 방식은 보안·플랫폼·하드웨어 연동처럼  
**문서만으로는 정답이 나오지 않는 영역**에서  
가장 빠르게 현실적인 해답에 도달하는 방법이라고 생각합니다.

---

## 🧭 개발 방식 (How I Work)

### 1) Top-down 이해
- 요구사항·목표·제약 조건을 **시스템 단위로 정리**
- 공격자 관점(침해 시나리오)과 운영자 관점(장애·모니터링)을 동시에 고려
- 모듈 경계와 책임을 먼저 정의하고, **향후 무너질 수 있는 지점**을 예측

### 2) Bottom-up 구현 및 검증
- 가장 불확실한 요소부터 **PoC로 우선 검증**
- 커널·네트워크·센서·프로토콜·데이터 흐름을 직접 확인
- 검증 결과를 반영해 설계를 수정하고, 다시 확장 가능한 형태로 고도화

### 3) 운영 가능한 구조로 정리
- 장애 대응, 로그, 관측 가능성(Observability)을 구조에 포함
- 데이터 무결성, 예외 처리, 권한·접근 제어 등 **운영 리스크를 설계 단계에서 반영**

## 🔐 보안 배경 (Security Background)

10년 이상 정보보안 현장에서 레드팀 활동과 침해사고 대응을 수행해왔습니다.  
시스템을 **공격자의 시선으로 분석**하고,  
그 결과를 **방어 가능한 구조로 설계에 반영**하는 것이 강점입니다.

- 레드팀: 모의 침투 테스트, 취약점 분석 및 대응
- 침해사고 대응: 로그·패킷 분석, 공격 패턴 식별, 사고 보고 및 복구
- 보안 설계: 접근 제어 정책 수립, 시스템 점검 및 개선

**활용 기술·도구**
- Kali Linux, Wireshark, Nmap, Metasploit
- Linux 커널·네트워크 분석, Python, eBPF

## 📌 대표 프로젝트 (Representative Projects)

> 모든 프로젝트는  
> **요구사항 정의 → 구조 설계 → PoC 검증 → 운영 가능한 형태로 정리**

### 1) ERP 시스템 개발 (2024.01 ~ 2024.05)
**역할**: 단독 개발 (요구사항 분석 → 설계 → 구현 → 배포)  
**성격**: 외부 의뢰 / 상용 ERP

**문제(Top-down)**
- 재고·주문·회계 요구사항이 복잡하고 변경이 잦음
- 성능 병목 및 데이터 일관성 문제가 발생할 가능성 존재

**접근(Bottom-up)**
- 대용량 트랜잭션 병목 구간을 우선 식별하고 쿼리·인덱스를 실측 기반으로 개선
- 실제 데이터 흐름을 기준으로 API 응답 구조를 반복 조정

**결과**
- 도메인 단위 분해로 **모듈화된 엔터프라이즈 로직** 구축
- RESTful API 기반 데이터 흐름 일관성 확보
- 인덱스 최적화 및 쿼리 리팩토링으로 **응답 속도 40% 이상 개선**

**기술 스택**: Java, Spring, JSP/Servlet, MySQL

- 센서·로봇·백엔드·웹 간 데이터 흐름 설계

**접근(Bottom-up)**
- 전체 시스템을 한 장의 구조로 정리
  - Sensor / Agent / Backend API / Web Monitoring Layer
- UWB 센서 데이터부터 실제 수집·신뢰도 검증
- 실시간 처리와 이력 저장 로직 분리

- 초기 혼란 상태의 구조를 **확장 가능한 시스템 구조로 정리**
- 로봇·센서·웹을 하나의 흐름으로 이해할 수 있는 기준 마련
- 모니터링 시스템을 **운영·디버깅 도구**로 활용 가능

**기술 스택**
- Java, Spring, REST API
- HTML, CSS, JavaScript, Chart.js
- DB 연동, UWB 센서 데이터 처리
